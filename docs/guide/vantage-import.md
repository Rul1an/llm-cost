# Vantage Import Guide (FOCUS v1.0)

This guide explains how to import `llm-cost` exports into [Vantage](https://vantage.sh) using the Custom Provider feature.

## Overview

The `llm-cost export` command generates a **FOCUS 1.0 (Billing Data)** compliant CSV file, specifically optimized for Vantage's strict validation rules.

```bash
llm-cost export --output costs.csv
```

## CSV Specifications

The export uses a subset of FOCUS columns guaranteed to work with Vantage.

| Column | Description | Source |
|---|---|---|
| **ChargePeriodStart** | Usage date (YYYY-MM-DD) | Defaults to `today` (UTC) |
| **ChargeCategory** | Type of charge | Static: `"Usage"` |
| **BilledCost** | Estimated cost in USD | `llm-cost` estimation |
| **ResourceId** | Unique identifier | `prompt_id` from manifest (or slug) |
| **ServiceName** | Service Name | Static: `"LLM Inference"` |
| **Tags** | Metadata | JSON object containing all extra fields |

### Tags & Metadata

Vantage allows filtering and breaking down costs by Tags. We encode all useful metadata here:

*   **`resource-name`**: The original file path (e.g., `prompts/search.txt`).
*   **`provider`**: The LLM provider (e.g., `OpenAI`).
*   **`model`**: The specific model used (e.g., `gpt-4o`).
*   **`team`**, **`app`**: User-defined tags from `llm-cost.toml`.
*   **`effective-cost`**: The exact cost (4 decimals).
*   **`x-token-count-*`**: Input/Output token metrics.

## Importing to Vantage

1.  Go to **Settings > cost Providers**.
2.  Select **Add Provider > Custom Provider**.
3.  Name it (e.g., "LLM Cost").
4.  Upload the `costs.csv` file generated by `llm-cost`.
5.  Wait for processing.

Once processed, you will see "LLM Inference" in your cost reports, and you can Group By "Tag: team" or "Tag: model".
